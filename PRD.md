# 司法鉴定意见书自动生成系统 - 产品需求文档 (PRD)

> 版本：v1.0  
> 日期：2026-04-19  
> 状态：已确认

---

## 一、项目背景

### 1.1 现状
- 鉴定人收到法院纸质材料后，需手动在 Word 模板中逐字输入鉴定意见书内容
- 基本情况、基本案情、资料摘要等内容来源于纸质材料，打字工作量大
- 人工输入容易出错，效率低

### 1.2 目标
- 上传材料图片 → OCR 自动识别 → 自动生成鉴定意见书各部分 → 鉴定人修正 → 生成正式报告
- 节省打字时间，减少人工输入错误
- 单机版，单用户

### 1.3 OCR 方案
- 使用硅基流动提供的 PaddleOCR 服务
- 识别率满足需求

| 角色 | 说明 |
|------|------|
| 鉴定人 | 主要使用者，上传材料、修正内容、确认并生成最终报告 |

## 三、核心流程

```
拍照/扫描 → 上传材料 → 点击「开始识别」→ OCR识别 → 自动生成报告各部分 → 鉴定人修正 → 确认完成 → 导出Word
```

---

## 四、案件状态

| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| 待上传 | 案件刚创建 | 上传材料、删除材料 |
| 识别中 | 材料上传完毕，正在OCR | 等待、查看进度 |
| 待修正 | OCR完成，内容已自动生成 | 查看识别结果、修正各部分内容 |
| 修正中 | 鉴定人正在修改 | 编辑各部分、补充鉴定过程/分析说明 |
| 待确认 | 所有内容填写完毕 | 确认鉴定意见、最终检查 |
| 已完成 | 报告确认无误，已生成 | 查看、导出Word、可重新打开 |

### 状态流转规则

1. **待上传 → 识别中**：鉴定人点击「开始识别」按钮触发 OCR
2. **识别中 → 待修正**：OCR 全部完成后自动流转
3. **待修正 → 修正中**：鉴定人第一次编辑任何内容时自动流转
4. **修正中 → 待确认**：鉴定人点击「提交审核」
5. **待确认 → 已完成**：鉴定人确认鉴定意见无误后点击「确认完成」
6. **已完成 → 修正中**：发现问题可「重新打开」

## 六、鉴定意见书结构及自动生成规则

### 6.1 基本情况

| 字段 | 来源 | 自动/手动 | 备注 |
|------|------|-----------|------|
| 委托单位 | 委托书 | 自动提取 | 委托书 title 位置 |
| 委托事项 | 委托书 | 自动提取 | |
| 受理日期 | — | **鉴定人手动输入** | |
| 鉴定材料清单 | 从材料清单提取 | 自动生成 | 逐条列出6类材料 |
| 鉴定日期 | = 受理日期 | 自动填入 | |
| 鉴定地点 | 固定值 | 自动填入 | 鉴定中心名称 |
| 在场人员 | — | 鉴定人填写 | 可空 |
| 被鉴定人信息 | 身份证 | 自动提取 | 姓名、性别、出生日期、身份证号、住址 |

**鉴定材料清单格式：**
```
鉴定材料：
1. 委托书壹份；
2. XXX居民身份证复印件壹份；
3. 道路交通事故认定书复印件壹份；
4. 鉴定申请书壹份；
5. XX医院住院病历复印件壹份；
6. CT片肆张
```

### 6.2 基本案情

| 字段 | 来源 | 备注 |
|------|------|------|
| 事故时间 | 委托书"简要案情" | |
| 事故地点 | 委托书"简要案情" | |
| 事故经过 | 委托书"简要案情" | |
| 就医经过 | 提取各医院就医情况 | 简要说明去哪几家医院就医 |
| 收尾句 | 自动生成 | 固定格式 |

**收尾句格式：**
> 现为处理案件需要，XXXX（委托单位）特委托我鉴定中心XXXXXX（委托事项）。

**注意：** 基本案情中不需要伤情摘要。

### 6.3 资料摘要

每个住院记录提取以下字段：

| 字段 | 说明 | 必需 |
|------|------|------|
| 医院名称 | 从病历识别 | ✅ |
| 住院号 | 从病历识别 | ✅ |
| 主诉 | 从病历识别 | ✅ |
| 现病史 | 从病历识别，必须记录清楚 | ✅ |
| 既往史 | 如有就记录，没有不反映 | ⚠️ 可选 |
| 体格检查 | 从病历识别 | ✅ |
| 影像学检查 | CT/X光报告文字 | ✅ |
| 入院诊断 | 从病历识别 | ✅ |
| 治疗过程 | 手术记录、用药情况（如有就记录） | ⚠️ 可选 |
| 出院诊断 | 从病历识别 | ✅ |
| 出院医嘱 | 从出院记录识别 | ✅ |

**多套病历处理：** 每个医院上传一套病历，多次住院上传多套，按住院号合并同一患者的多次住院记录。

### 6.4 鉴定过程

| 段落 | 内容 | 自动/手动 |
|------|------|-----------|
| ① 第一段 | 格式语言（固定模板） | 自动生成 |
| ② 第二段 | 鉴定人填写 | 手动 |
| ③~N 后续段 | 影像学复阅（有几个片子生成几段） | 自动生成 |

**第一段格式语言：**
> 按照《法医临床检验规范》（SF/Z JD0103003-2011)和《法医临床影像学检验实施规范》（SF/Z JD0103006-2014）对被鉴定人XXX（姓名）进行法医临床学检验。

**影像学复阅格式（每个片子一段）：**
> 复阅XXXX年X月X日（报告日期）XXX医院XXX（被鉴定人）XXX（片子名称）（号XXXX（片子编号））示：XXXX（影像学报告内容）

### 6.5 分析说明

| 序号 | 内容 | 自动/手动 |
|------|------|-----------|
| ① | 外伤史确切 | 自动生成 |
| ② | 诊断明确 | 自动生成（根据实际资料判断写X线/CT/临床检查及手术） |
| ③ | 委托鉴定事项草稿 | 自动生成草稿，鉴定人修改 |

**① 格式：**
> 1. 被鉴定人XXX外伤史确切，系道路交通所致。

**② 格式：**
> 2. 被鉴定人XXX XXX（病情摘要）等诊断明确（经X线（如有）、CT、临床检查及手术证实）。

**③ 格式：** 根据委托事项生成伤残等级、误工期、后续治疗费等草稿内容。

**📝 风格学习：** 系统记录鉴定人的修改习惯，下次生成更贴合个人风格。

### 6.6 鉴定意见

- 根据分析说明自动生成草稿
- **鉴定人确认无误后才能使用**（关键安全机制）
