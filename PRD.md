# 司法鉴定意见书自动生成系统 - 产品需求文档 (PRD)

> 版本：v1.1  
> 日期：2026-04-22  
> 状态：已完成开发

---

## 一、项目背景

### 1.1 现状
- 鉴定人收到法院纸质材料后，需手动在 Word 模板中逐字输入鉴定意见书内容
- 基本情况、基本案情、资料摘要等内容来源于纸质材料，打字工作量大
- 人工输入容易出错，效率低

### 1.2 目标
- 上传材料图片 → OCR 自动识别 → 自动生成鉴定意见书各部分 → 鉴定人修正 → 生成正式报告
- 节省打字时间，减少人工输入错误
- 单机版，单用户

### 1.3 OCR 方案
- 使用硅基流动提供的 PaddleOCR 服务
- 识别率满足需求

## 三、核心流程

```
拍照/扫描 → 上传材料 → 点击「开始识别」→ OCR识别 → 自动生成报告各部分 → 鉴定人修正 → 确认完成 → 导出Word
```

---

## 四、案件状态

| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| 待上传 | 案件刚创建 | 上传材料、删除材料 |
| 识别中 | 材料上传完毕，正在OCR | 等待、查看进度 |
| 待修正 | OCR完成，内容已自动生成 | 查看识别结果、修正各部分内容 |
| 修正中 | 鉴定人正在修改 | 编辑各部分、补充鉴定过程/分析说明 |
| 待确认 | 所有内容填写完毕 | 确认鉴定意见、最终检查 |
| 已完成 | 报告确认无误，已生成 | 查看、导出Word、可重新打开 |

### 状态流转规则

1. **待上传 → 识别中**：鉴定人点击「开始识别」按钮触发 OCR
2. **识别中 → 待修正**：OCR 全部完成后自动流转
3. **待修正 → 修正中**：鉴定人第一次编辑任何内容时自动流转
4. **修正中 → 待确认**：鉴定人点击「提交审核」
5. **待确认 → 已完成**：鉴定人确认鉴定意见无误后点击「确认完成」
6. **已完成 → 修正中**：发现问题可「重新打开」

## 六、鉴定意见书结构及自动生成规则

### 6.1 基本情况

| 字段 | 来源 | 自动/手动 | 备注 |
|------|------|-----------|------|
| 委托单位 | 委托书 | 自动提取 | 委托书 title 位置 |
| 委托事项 | 委托书 | 自动提取 | |
| 受理日期 | — | **鉴定人手动输入** | |
| 鉴定材料清单 | 从材料清单提取 | 自动生成 | 逐条列出7类材料；民事起诉状与鉴定申请书合并列项 |
| 鉴定日期 | = 受理日期 | 自动填入 | |
| 鉴定地点 | 固定值 | 自动填入 | 鉴定中心名称 |
| 在场人员 | — | 鉴定人填写 | 可空 |
| 被鉴定人信息 | 身份证 | 自动提取 | 姓名、性别、出生日期、身份证号、住址 |

**鉴定材料清单格式：**
```
鉴定材料：
1. 委托书壹份；
2. XXX居民身份证复印件壹份；
3. 道路交通事故认定书复印件壹份；
4. 民事起诉状、鉴定申请书复印件各壹份；
5. XX医院住院病历复印件壹份；
6. CT片肆张
```

### 6.2 基本案情

| 字段 | 来源 | 备注 |
|------|------|------|
| 事故时间 | 委托书"简要案情" | |
| 事故地点 | 委托书"简要案情" | |
| 事故经过 | 委托书"简要案情" | |
| 就医经过 | 提取各医院就医情况 | 简要说明去哪几家医院就医 |
| 收尾句 | 自动生成 | 固定格式 |

**收尾句格式：**
> 现为处理案件需要，XXXX（委托单位）特委托我鉴定中心XXXXXX（委托事项）。

**注意：** 基本案情中不需要伤情摘要。

### 6.3 资料摘要

每个住院记录提取以下字段：

| 字段 | 说明 | 必需 |
|------|------|------|
| 医院名称 | 从病历识别 | ✅ |
| 住院号 | 从病历识别 | ✅ |
| 主诉 | 从病历识别 | ✅ |
| 现病史 | 从病历识别，必须记录清楚 | ✅ |
| 既往史 | 如有就记录，没有不反映 | ⚠️ 可选 |
| 体格检查 | 从病历识别 | ✅ |
| 影像学检查 | CT/X光报告文字 | ✅ |
| 入院诊断 | 从病历识别 | ✅ |
| 治疗过程 | 手术记录、用药情况（如有就记录） | ⚠️ 可选 |
| 出院诊断 | 从病历识别 | ✅ |
| 出院医嘱 | 从出院记录识别 | ✅ |

**多套病历处理：** 每个医院上传一套病历，多次住院上传多套，按住院号合并同一患者的多次住院记录。

### 6.4 鉴定过程

| 段落 | 内容 | 自动/手动 |
|------|------|-----------|
| ① 第一段 | 格式语言（固定模板） | 自动生成 |
| ② 第二段 | 鉴定人填写 | 手动 |
| ③~N 后续段 | 影像学复阅（有几个片子生成几段） | 自动生成 |

**第一段格式语言：**
> 按照《法医临床检验规范》（SF/Z JD0103003-2011)和《法医临床影像学检验实施规范》（SF/Z JD0103006-2014）对被鉴定人XXX（姓名）进行法医临床学检验。

**影像学复阅格式（每个片子一段）：**
> 复阅XXXX年X月X日（报告日期）XXX医院XXX（被鉴定人）XXX（片子名称）（号XXXX（片子编号））示：XXXX（影像学报告内容）

### 6.5 分析说明

| 序号 | 内容 | 自动/手动 |
|------|------|-----------|
| ① | 外伤史确切 | 自动生成 |
| ② | 诊断明确 | 自动生成（根据实际资料判断写X线/CT/临床检查及手术） |
| ③ | 委托鉴定事项草稿 | 自动生成草稿，鉴定人修改 |

**① 格式：**
> 1. 被鉴定人XXX外伤史确切，系道路交通所致。

**② 格式：**
> 2. 被鉴定人XXX XXX（病情摘要）等诊断明确（经X线（如有）、CT、临床检查及手术证实）。

**③ 格式：** 根据委托事项生成伤残等级、误工期、后续治疗费等草稿内容。

**📝 风格学习：** 系统记录鉴定人的修改习惯，下次生成更贴合个人风格。

### 6.6 鉴定意见

- 根据分析说明自动生成草稿
- **鉴定人确认无误后才能使用**（关键安全机制）

## 七、功能特性

| 功能 | 说明 |
|------|------|
| 网页在线编辑 | 所有内容可在网页修改 |
| OCR 识别修正 | 识别有误可手动修正 |

| 风格学习 | 记录鉴定人修改习惯，优化后续生成 |
| Word 导出 | 最终生成可打印的 Word 报告 |
| 案件状态管理 | 6个状态追踪案件进度 |
| 单机版 | 本地运行，无需服务器部署 |

### 7.1 PDF 导入功能详解

**使用场景：** 鉴定人收到的是 PDF 格式的电子材料，而非纸质照片。

**操作流程：**
1. 在上传材料页面点击「PDF 导入」
2. 选择 PDF 文件，点击「转换」
3. 系统自动调用 `pdftoppm` 将 PDF 分割为 PNG 图片
4. 以缩略图网格形式展示所有页面
5. 点击缩略图可弹窗查看大图
6. 勾选需要的页面，选择目标分类（如「身份证」「病历」），点击「导入」
7. 未使用的页面可选中后删除

**技术实现：**
- 后端：`pdftoppm -png -r 150` 命令（需安装 poppler）
- 前端：缩略图网格 + 全屏预览对话框
- 存储：`uploads/{case_id}/pdf_pages/` 目录

## 八、材料分组机制

### 8.1 分组类型

| 类型 | 上传方式 | 说明 |
|------|----------|------|
| 委托书 | 多文件平铺 | 每张单独上传，不分组 |
| 身份证 | 多文件平铺 | 正面+反面各一张 |
| 交通事故认定书 | 多文件平铺 | 可能多页 |
| 鉴定申请书 | 多文件平铺 | 可能多页 |
| **医院病历** | **按医院分组** | 一个医院=一个分组，组内多页 |
| **影像学报告** | **按医院分组** | 同上 |

### 8.2 分组操作
- 新增分组：输入医院名称（如「新乡市中心医院」）
- 重命名分组：点击「改名」按钮
- 删除分组：删除分组及其中所有文件
- 分组内上传：支持单张/批量上传

## 九、LLM 智能提取

### 9.1 可提取内容

| 提取类型 | 数据来源 | 输出目标 |
|----------|----------|----------|
| 被鉴定人信息 | 身份证 | persons 表 |
| 委托信息 | 委托书 | cases.entrusting_unit, entrustment_matter |
| 案情信息 | 交通事故认定书 | reports.case_facts |
| 住院记录 | 病历材料 | hospital_records 表 |
| 影像报告 | 影像材料 | imaging_reports 表 |

### 9.2 可生成内容

| 生成类型 | 说明 |
|----------|------|
| 基本案情 | 从委托书和住院记录提取事故经过和就医经过 |
| 资料摘要 | 从住院记录生成规范摘要文本 |
| 鉴定过程 | 固定格式 + 影像复阅段落 |
| 分析说明 | 外伤史 + 诊断 + 委托事项草稿 |
| 鉴定意见 | 根据分析说明生成草稿 |

### 9.3 风格学习

系统记录鉴定人对 AI 生成内容的修改：
- 保存原文、修改后文本、差异摘要
- 积累后用于优化后续生成风格
- 存储在 `style_logs` 表
